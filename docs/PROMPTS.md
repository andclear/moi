# 《来岛上 / MOI》提示词文档

本文档记录项目提示词策略。阶段 0 仅建立提示词文档入口，具体提示词会随 LLM 适配层与业务步骤逐步实现。

## 通用原则

- 所有 AI 调用必须经过统一 LLM 适配层。
- 提示词必须区分“用户确认事实”“已锁定事实”和“AI 推测内容”。
- 不允许业务页面直接拼接上游请求。
- 兼容不支持 `system` prompt 的 OpenAI 兼容接口。
- 所有最终面向用户保存或导出的角色卡内容使用简体中文；后续如果增加 YAML 导出，YAML 键名和值也优先使用中文表达，不使用英文模板字段直接外显。
- 叙事字段中使用 `{{user}}` 指代用户，使用 `{{char}}` 指代角色。

## 侧写提示词

- `src/prompts/profilePrompts.ts` 集中维护寻人启事与侧写四段式互动提示词。
- 初始侧写将自然语言描述整理为 Markdown 角色档案。
- 侧写四段式互动分别生成轮廓、反例、叙事碎片和日记真相候选。
- 输出必须为可解析 JSON，由 `src/schemas/llmResponseSchemas.ts` 校验。
